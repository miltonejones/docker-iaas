## Response

## Final Analysis

### What happened

The `.offcanvas-session-row__delete` button rendered in `App.tsx:501` carries the classes `btn btn--ghost btn--sm offcanvas-session-row__delete`. The `.btn` base class applies `border: 1px solid var(--border-strong)`, giving all buttons a visible border — including this delete button, where a border is unwanted.

### The fix (already applied as a working-tree change)

The committed code (on GitHub `main`) only had:

```css
.offcanvas-session-row__delete {
  flex-shrink: 0;
}
```

The working tree (`M web/src/styles.css`) adds three things:

| Line | Addition | Purpose |
|------|----------|---------|
| 2901 | `border: none !important;` | Removes the border inherited from `.btn` and `.btn--ghost` |
| 2902 | `outline: none;` | Removes the browser focus outline for a clean appearance |
| 2904-2906 | `border: none !important;` on `:hover:not(:disabled)` | Prevents `.btn:hover:not(:disabled)` (specificity 0,2,0) from re-introducing a `border-color` on hover |

### Why `!important` matters

Without `!important`, the hover state fix would still work (`border-style: none` persists regardless of `border-color` changes), but `!important` provides a definitive, unambiguous override that won't break if new `.btn` variant rules are added later.

### Result

No additional edits are needed — the fix is already present in the working tree. The delete button renders without a border in both normal and hover states.
