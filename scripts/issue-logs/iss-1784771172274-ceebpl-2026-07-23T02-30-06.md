## Summary of Changes

### Root Cause
The assistant had no text-to-speech capability — responses were only displayed as text in the chat UI. There was no mechanism to speak responses aloud, and no toggle to control such a feature.

### Changes Made

#### 1. `web/src/icons.tsx` — Added volume/speaker icons
- Imported `faVolumeHigh` and `faVolumeXmark` from `@fortawesome/free-solid-svg-icons`
- Registered them as `'volume-high'` and `'volume-mute'` in the `UI_ICONS` map so they're available to the `AppIcon` component

#### 2. `web/src/components/AssistantBar.tsx` — Added speech TTS integration and toggle

**State & persistence:**
- Added `SPEECH_KEY = 'dockyard:assistant-speech'` — localStorage key for persisting the toggle across sessions/refreshes
- Added `speechEnabled` state, initialized from `localStorage` so the preference survives refreshes
- Added a `useEffect` that writes the preference to `localStorage` whenever it changes

**TTS engine:**
- Added `stripMarkdownForSpeech()` — strips markdown formatting (code blocks, links, bold/italic, headings, blockquotes, HTML tags, etc.) so the text reads naturally when spoken aloud
- Added `speakText()` — uses the browser's built-in `window.speechSynthesis` (Web Speech API) to speak assistant response text. It cancels any ongoing speech first to prevent stacking, and includes a guard for browsers without Speech Synthesis support

**Speech trigger:**
- Calls `speakText(streamedText || turn.text)` inside `consumeTurnStream()` when the turn completes — so the finalized assistant response is spoken automatically when speech is enabled

**Toggle button:**
- Added a toggle button in the `assistant-session-bar__actions` row (alongside "New" and "Sessions" buttons)
- Shows `faVolumeHigh` icon with "Voice on" label when enabled (highlighted with `btn--primary` style)
- Shows `faVolumeXmark` icon with "Voice off" label when disabled
- Includes `title` and `aria-label` for accessibility
