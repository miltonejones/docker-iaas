## Diagnosis

The issue had two parts:

1. **Wrong order in the footer** — The rotating slogan (`FooterSlogan`) was rendered *before* the "Create an issue" button inside `app-footer__group`, placing the slogan on the left and the button on the right. The issue requires the slogan to be to the right of "Create an issue".

2. **No transition animation** — The `FooterSlogan` component used `useState` to swap the text every 8 seconds, but the `<span>` had no `key` prop, so React just mutated the text content of the same DOM node with no way to trigger a CSS transition.

## Changes Made

### `web/src/App.tsx`
- **Swapped the order** in the footer: the "Create an issue" button now comes first (left), and `<FooterSlogan />` follows it (right).
- **Added `key={slogan}` and a `footer-slogan` class** to the `<span>` in `FooterSlogan`. The changing `key` causes React to unmount/remount the element on each slogan change, which re-triggers the CSS animation. Also removed the generic `muted` class since the new `.footer-slogan` class handles the muted color.

### `web/src/styles.css`
- **Added `.footer-slogan`** — sets `display: inline-block`, `color: var(--muted)`, and the `reveal-right` animation.
- **Added `@keyframes reveal-right`** — animates `clip-path` from `inset(0 100% 0 0)` (fully hidden, clipped from the right) to `inset(0 0 0 0)` (fully visible). This creates a rightward reveal where the text is progressively uncovered from left to right over 0.5 seconds.
