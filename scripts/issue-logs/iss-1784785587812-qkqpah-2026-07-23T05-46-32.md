## Summary

### Root Cause

When new code is deployed (via the CI/CD pipeline that runs `docker compose up --build -d` on pushes to `main`), the server restarts with updated code. The client-side SPA — already loaded in the user's browser — has no way to know that a new version is available. The user continues using stale client assets (JS/CSS) until they manually refresh. 

The notification pipeline already generates deploy-related events — the CI workflow writes `✅ Deployed` to `notifications.jsonl`, and the SSE stream delivers these to connected browsers — but the frontend had no logic to act on them.

### Changes Made

**`web/src/components/NotificationBell.tsx`:**
- Added `isDeployNotification()` — detects deploy notifications by matching `🚀` prefix, `📤` prefix (push notification from the issue consumer), or summaries containing `"deploy"`/`"deployed"` (catches CI's `✅ Deployed`)
- Added `NotificationBellProps` interface with two optional callbacks: `onDeployNotification` (fires on explicit deploy notifications) and `onStreamReconnect` (fires when the SSE stream reconnects, indicating a possible server restart)
- The `notify` callback now checks incoming notifications against `isDeployNotification()` and calls `onDeployNotification` when matched
- The SSE history handler now calls `onStreamReconnect` when the stream reconnects after being established
- In-memory deduplication (`reloadPrompted` ref) ensures the user gets only one prompt per deploy cycle

**`web/src/App.tsx`:**
- Added `handleDeployNotification` callback that shows a `window.confirm()` dialog with the deploy notification summary and "Reload the page to pick up the latest changes?" — calls `window.location.reload()` if confirmed
- Wired both `onDeployNotification` and `onStreamReconnect` props on `<NotificationBell>` to the handler

**Two detection paths cover different scenarios:**
1. **Explicit notification** — CI writes `✅ Deployed` to the notification log → SSE delivers it → reload prompt
2. **SSE reconnect** — server restarts with new code → browser's EventSource reconnects → reload prompt
