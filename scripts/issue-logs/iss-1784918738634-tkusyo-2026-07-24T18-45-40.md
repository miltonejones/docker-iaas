## Diagnosis

The `mono` CSS class (monospace font stack: JetBrains Mono / SF Mono / ui-monospace) was applied to the **first column** of six entity list/table views. These first columns display human-readable names (function names, bucket names, issue summaries, table/collection names) that don't benefit from monospace rendering.

The Instances table and Gateway table already rendered their first columns without `mono` — these six were inconsistent outliers.
