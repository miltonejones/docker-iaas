## Summary

**Root cause:** The `GatewayList` component had a `viewMode` state that defaulted to `'table'` on every mount. Unlike the `Instances` component (which already persisted its list/grid preference in `localStorage` under `instances-view`), the gateway list had no persistence — so switching to grid view was lost whenever the user navigated away and back.

**Fix:** Added the same `localStorage` persistence pattern from `Instances.tsx` into `GatewayPanel.tsx`:

1. **Added** `VIEW_KEY = 'gateway-view'` constant and `ViewMode` type (lines 11-12)
2. **Added** `loadView()` helper that reads from `localStorage`, defaulting to `'table'` with a try/catch guard for private browsing (lines 14-21)
3. **Changed** `useState` initializer from the hardcoded `'table'` to `loadView` (line 40)
4. **Added** `changeView()` helper that both updates state and writes to `localStorage` (lines 42-49)
5. **Updated** both toggle button `onClick` handlers from `setViewMode` to `changeView` (lines 121, 128)

Now when a user switches the gateway list to grid view, that choice persists across page navigations and browser sessions.
