## Diagnosis

**Root cause**: The `FooterSlogan` component used `key={slogan}` to force React to unmount/remount the `<span>` on each rotation. When the slogan changed, the old `<span>` was **instantly destroyed** by React — only the new element received the `reveal-right` clip-path animation. There was no exit animation for the old text. The result was a jarring instant cut where the old text simply vanished.

**What "Clip-Path Inset Reveal" means here**: A proper two-phase reveal where the outgoing text clips _out_ (e.g., slides left) while the incoming text clips _in_ (e.g., slides right from behind it), with both elements visible during the crossover. This gives a smooth, polished rotation effect using `clip-path: inset()`.

## Changes

### `web/src/App.tsx`
Rewrote the `FooterSlogan` component:
- Tracks the previous slogan in state (`prevSlogan`) so it stays in the DOM during the exit animation
- Uses a `cancelled` flag and ref guards so timeouts clean up correctly on unmount
- Renders both old and new text overlapping inside a wrapper via CSS grid stacking
- Old text: `.footer-slogan--exit` — plays a 400ms clip-out animation
- New text: `.footer-slogan--enter` + dynamic `key` — plays a 550ms clip-in animation after a 400ms delay
- Timing: exit (400ms) → swap → enter (550ms) → cleanup

### `web/src/styles.css`
Replaced the single `.footer-slogan` animation:
- **`.footer-slogan-wrap`** — `display: inline-grid` stacks child spans on the same grid cell so they overlap without affecting layout
- **`.footer-slogan--enter`** / **`@keyframes slogan-clip-in`** — `inset(0 100% 0 0)` → `inset(0 0 0 0)`, text reveals from right
- **`.footer-slogan--exit`** / **`@keyframes slogan-clip-out`** — `inset(0 0 0 0)` → `inset(0 0 0 100%)`, text retreats to the left

The net effect: old slogan slides out left, new slogan slides in right — a proper two-phase clip-path inset reveal.
