## Summary of Changes

### Diagnosis

The existing TTS implementation had two issues:

1. **Per-message-only UX**: Speaker buttons only appeared on hover over individual assistant messages. The user had to manually click each one to hear it read aloud — no automatic behavior.

2. **Wrong role for the volume icon**: The volume icon (`faVolumeHigh`) in the session bar opened a voice selector dropdown but didn't control whether messages were spoken. It was a voice picker, not a global on/off toggle.

The user wanted a hands-free experience: a global toggle that, when ON, auto-speaks every assistant message as it completes.

### Changes Made

**`/app/web/src/components/AssistantBar.tsx`** — 5 edits:

1. **Added global TTS state** (line 286-289): New `globalSpeakEnabled` state initialized from `localStorage` key `dockyard:tts-global-speak` (defaults ON), plus a ref (`globalSpeakRef`) so async callbacks can read the latest value, and `lastAutoSpokenIdxRef` to track which messages have been auto-spoken.

2. **Persistence effect** (line 297-302): Writes the preference to `localStorage` whenever `globalSpeakEnabled` changes, and keeps the ref in sync.

3. **Auto-speak effect** (line 389-406): A `useEffect` on `[busy]` that fires when a turn completes (`busy` transitions `true` → `false`). If global speak is ON, it finds the most recent assistant message that hasn't been auto-spoken yet and calls `speakMessage()` via `requestAnimationFrame`.

4. **Session bar UI** (line 1390-1408): Repurposed the volume icon as a global toggle button:
   - ON state: shows `speak` icon (`faVolumeHigh`) in accent color with accent background
   - OFF state: shows `mute` icon (`faVolumeXmark`) in muted color
   - Clicking OFF while speaking calls `stopSpeaking()` to cancel current audio
   - The voice selector is now a separate small chevron-down button next to the toggle

5. **Reset tracking on session changes**: `lastAutoSpokenIdxRef` is reset in `resetToNewSession()`, `loadSession()`, and `restoreFallback()` so loaded/recovered history isn't re-spoken.

**`/app/web/src/styles.css`** — 1 edit:

6. **CSS for toggle states**: Added `.assistant-tts-toggle` (muted default), `.assistant-tts-toggle--on` (accent-colored), and `.assistant-voice-select__trigger` (compact voice picker button). Also made `.assistant-voice-select` a flex container to lay out the toggle + voice trigger side by side.

### Existing UI Preserved

- Per-message speaker buttons (appear on hover) remain as a secondary override
- Voice selector dropdown still lets the user choose their preferred synthesis voice
- Copy button and all other message controls are untouched
- All existing TTS helpers (`speakMessage`, `stopSpeaking`) are unchanged
