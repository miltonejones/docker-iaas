## Response

## Summary

### Diagnosis

The assistant (`AssistantBar.tsx`) only rendered text responses via `ReactMarkdown` in the chat UI. There was no text-to-speech integration, no speech toggle button, and no mechanism to persist a user's speech preference. This was a missing feature — the codebase simply never had TTS support wired up.

### Root Cause

A missing feature — the assistant component had no Web Speech API integration, no toggle UI, and no persistence for speech preference.

### Changes Made

**3 files modified:**

1. **`web/src/icons.tsx`** — Added `faVolumeHigh` and `faVolumeXmark` icons from FontAwesome, mapped to `'volume-high'` and `'volume-off'` in the `UI_ICONS` dictionary.

2. **`web/src/components/AssistantBar.tsx`** — Core feature implementation:
   - Added `speechEnabled` state (initialized from `localStorage` key `dockyard:assistant-speech`)
   - Added `stripMarkdownForSpeech()` — strips code blocks, inline code, links, headings, bold/italic, list markers, and blockquotes so TTS reads cleanly
   - Added `speakAssistantText()` — cancels any ongoing speech, cleans the text, and speaks it via `window.speechSynthesis` (Web Speech API), with a try/catch for environments where speech synthesis is unavailable
   - Added `lastSpokenIndexRef` to track which assistant log entries have already been spoken, preventing repeats
   - Added a `useEffect` that watches `log` and `speechEnabled` to speak new assistant entries as they appear
   - Added a `toggleSpeech()` handler that flips the state and persists to `localStorage`
   - Added a speech toggle button in the `assistant-session-bar__actions` group (between "Sessions" and the end), showing either a volume-high icon ("Speaking") or volume-off icon ("Mute")
   - Updated `resetToNewSession()`, `loadSession()`, `restoreFallback()`, and the initial session-loading effect to reset `lastSpokenIndexRef` so old messages aren't re-spoken when switching sessions

3. **`web/src/styles.css`** — Added `.btn--speech-on` styles: accent-colored border, text, and background when speech is enabled, with hover state matching the design system.
