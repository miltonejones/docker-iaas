## Response

## Summary

### Root cause

The issues list used completely custom, unstyled HTML classes (`issues-page`, `issues-card`, `issues-list`, `issues-card__top`, etc.) that had **zero CSS rules** defined anywhere in `styles.css`. The list rendered as a stack of raw `<button>` elements with browser-default styling — no borders, no backgrounds, no hover states — which is why it didn't look like any other list in the app.

Other list pages (Functions, Databases, Gateway, Buckets) all use a consistent pattern: `<section className="panel">` → `<div className="panel__head">` → `<table className="table">`. The issues page had been written with a custom card-based layout that was completely detached from the design system.

The CSS that *did* exist for issues (tab bar, badges, consumer bar, issue detail) also had multiple problems:
- Used `rem` units instead of the project's `px`-based system
- Referenced `--primary` which is **not defined** in the `:root` — the project uses `--accent`
- The `@keyframes pulse` was **redefined** at line 3641 (opacity-based for consumer dot), clobbering the original box-shadow-based pulse (line 448) used by the live gauge indicator
- Used hardcoded color hex values (`#9ca3af`, `#10b981`, `#ef4444`) instead of design-system variables (`--down`, `--up`, `--crit`)
- The "New Issue" button navigated to `/issues/new` which has no route handler and renders a broken issue detail page

### Changes made

**`web/src/pages/IssuesList.tsx`** — Rewrote the issues list from the unstyled card layout to the standard panel + table pattern:
- Wrapped in `<section className="panel">` with `<div className="panel__head">` (matching Functions, Databases, etc.)
- Replaced unstyled `<button className="issues-card">` elements with standard `<table className="table">` rows
- Columns: Summary, Category (as a chip), Status (as a badge), Reported (time ago)
- Fixed "New Issue" button: calls `onCreateIssue()` (opens the `CreateIssueModal`) instead of navigating to the nonexistent `/issues/new` route
- Kept the tab bar for status filtering and the consumer status bar

**`web/src/styles.css`** — Fixed the issues CSS block (lines 3628–3661):
- **Tab bar**: Changed `rem` → `px` units, `--primary` → `--accent`, added hover state, used `var(--r-sm)` for border-radius
- **Badges**: Changed `rem` → `px` units
- **Consumer bar**: Changed `rem` → `px` units, dot colors use `var(--up)` / `var(--down)` instead of hardcoded hex
- **`@keyframes pulse`** renamed to `pulse-opacity` to stop it from clobbering the gauge's box-shadow pulse animation
- **Issue detail**: Changed all `rem` → `px` units, removed hardcoded `#ddd` fallbacks, used `var(--up)` / `var(--crit)` / `var(--down)` for activity timeline dots
