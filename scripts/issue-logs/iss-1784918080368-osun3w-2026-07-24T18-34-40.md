## Root Cause

The `Instances` component (`web/src/components/Instances.tsx`) had a hardcoded page size of 10 with no filtering capability. Users had no way to search or narrow down the container list, and the page size was double what was desired.

## Changes Made

**File: `web/src/components/Instances.tsx`**

1. **Page size reduced from 10 → 5** (line 60) — matches the requested page size.

2. **Client-side text filter added** (lines 47–57) — a new `filter` state variable drives case-insensitive matching against container `name`, `image`, `description`, and `state`. The filtered set feeds both the count badge and the pagination.

3. **Filter input UI** (lines 135–146) — a text input using the existing `.filter-bar` CSS class appears below the panel header whenever there are containers. Placeholder reads "Filter by name, image, description…".

4. **Dynamic count badge** (lines 99–107) — shows the filtered count, with a trailing "of N" hint when filtering is active and the filtered count differs from the total.

5. **No-match empty state** (line 150–151) — displays "No instances match filter …" when the filter yields zero results, distinct from the "No instances yet" state.

6. **Page resets on filter change** (line 71) — pagination snaps back to page 0 when the filter text or container count changes, preventing stale "page beyond bounds" states.
