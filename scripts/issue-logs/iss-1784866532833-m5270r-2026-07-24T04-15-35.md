## Summary

**Root cause:** The `Breadcrumbs` component in `App.tsx` rendered only text labels without any icons, even though the `SERVICES` array already mapped each service to an icon, and the `AppIcon` component was already imported and used elsewhere (in `ServiceNav`). This meant breadcrumb elements lacked the visual icons that would help users quickly identify where they are in the navigation hierarchy.

**What changed:**

1. **`web/src/App.tsx`** — Added `<AppIcon>` before each breadcrumb element's text:
   - **Home** link now shows a house icon (`home`)
   - **Service label** (e.g., "Instances", "Functions") now shows the service's icon from `SERVICES` (e.g., `container`, `function`)
   - **Detail label** (e.g., a specific container name, function name) now shows the parent service's icon for context

2. **`web/src/styles.css`** — Added `.breadcrumbs__icon` with `margin-right: 4px` to ensure consistent spacing between each icon and its adjacent text label.

Example: `🏠 Home > 📦 Instances > 📦 my-container` instead of the previous `Home > Instances > my-container`.
