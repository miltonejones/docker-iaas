## Summary

### Root Cause
The saved sessions sidebar (left offcanvas panel) in `App.tsx` rendered all sessions as a flat, ungrouped list. Each `AssistantSessionSummary` has `createdAt` and `updatedAt` ISO date fields, and the server returns them sorted by `updated_at DESC`, but no client-side grouping by date was applied — sessions just appeared in a single list regardless of when they were last active.

### Changes Made

**`/app/web/src/App.tsx`** — Two changes:

1. **Added two utility functions** (lines 76-107):
   - `dateGroupLabel(iso)` — categorizes an ISO date string into one of four buckets: `"Today"`, `"Yesterday"`, `"This Week"` (within 7 days), or `"Earlier"`. It compares calendar dates (ignoring time-of-day) so DST transitions and clock skew don't cause misclassification.
   - `groupSessionsByDate(sessions)` — partitions the session list by `dateGroupLabel` on each session's `updatedAt`, sorts each group's items newest-first, and returns an ordered array of `{ label, items }` objects.

2. **Changed the rendering** (lines 522-546): Replaced `sessionsList.map(...)` with `groupSessionsByDate(sessionsList).map(...)`, wrapping each group in a `<div>` with a `.offcanvas-session-group-header` element showing the group label (e.g. "Today", "Yesterday", "This Week", "Earlier").

**`/app/web/src/styles.css`** — Added styles for `.offcanvas-session-group-header` (lines 2926-2933): compact uppercase section header with muted color, matching the existing sidebar typography conventions (11px, 600 weight, muted color, letter-spacing).
