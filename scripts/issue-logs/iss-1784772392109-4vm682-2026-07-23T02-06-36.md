## Response

## Summary

### Root Cause

The consumer container couldn't push to GitHub because of **two intertwined issues**:

1. **Git "dubious ownership" error**: The consumer container bind-mounts `/app/.git` from the host (`./.git:/app/.git:rw`). The container runs as UID `1000:1000`, but the host's `.git` directory is owned by a different UID (the host user). Git ≥2.35.2 refuses to operate on repositories owned by a different user, so every git command — even `git diff --stat` — fails with `fatal: detected dubious ownership in repository at '/app'`. The consumer's `pushToGitHub()` function catches this error and silently logs "Could not detect changes — skipping push", so fixes are never committed or pushed.

2. **The consumer tried to fix itself but couldn't persist the fix**: The consumer correctly diagnosed this issue (see log `iss-1784771776754-5nb06t`) and wrote code changes to `docker-compose.yml` to add `git config --global --add safe.directory /app`. But because of the same dubious ownership bug, the consumer couldn't push the fix to GitHub. Even worse, the consumer container's `/app` directory isn't mounted from the host (only `.git`, `data`, and `scripts/issue-logs` are), so when the container restarts, any in-container file modifications are lost.

### Changes Made

**`docker-compose.yml`** (lines 57–84): Changed the consumer service configuration:

1. **Removed** `dockerfile: Dockerfile.consumer` — the file doesn't exist in the working tree and `/app` is not writable for new file creation (only existing file modification works in this overlay filesystem). The build now uses the main `Dockerfile` which provides Node.js + git + codebase as a superset.

2. **Updated the `command`** from a simple exec-form array to a multi-line shell script that:
   - Runs `git config --global --add safe.directory /app` — this is **the critical fix** that tells git to trust the `/app` directory regardless of UID mismatch
   - Checks if Claude Code CLI exists at `/usr/local/bin/claude`, and installs it via `npm install -g @anthropic-ai/claude-code` if missing (needed since the main Dockerfile doesn't include it)
   - Uses `exec node scripts/issue-consumer.mjs` to properly replace the shell process

### What still needs doing

1. Restore `Dockerfile.consumer` to the repo at `/app/Dockerfile.consumer` (a reference copy exists at `/home/node/Dockerfile.consumer`)
2. Switch the consumer build back to `dockerfile: Dockerfile.consumer` for a lighter image that doesn't include Playwright and the full web build
