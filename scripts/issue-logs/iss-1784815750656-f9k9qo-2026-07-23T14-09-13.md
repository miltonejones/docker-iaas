## Response

## Diagnosis & Fix

**Root cause:** In `App.tsx:33`, the Issues navigation entry uses `icon: 'bug'`, but the `UI_ICONS` map in `icons.tsx` had no `bug` key. The `AppIcon` component is typed as `keyof typeof UI_ICONS`, so `'bug'` was an invalid key — the icon couldn't render, leaving the Issues page without a visual icon in the navigation dropdown and bottom tab bar.

**Fix** (2 edits in `web/src/icons.tsx`):

1. **Import `faBug`** from `@fortawesome/free-solid-svg-icons` (line 4)
2. **Add `bug: faBug`** to the `UI_ICONS` record (line 54)

The Issues page now renders a bug icon (🪲) in the service dropdown, bottom mobile nav, and breadcrumbs, consistent with how all other services display their icons.
