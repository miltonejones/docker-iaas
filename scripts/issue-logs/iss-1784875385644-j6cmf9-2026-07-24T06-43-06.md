## Root Cause

The consumer's AI model (Claude/DeepSeek) **accidentally deleted `Dockerfile.consumer` and `.gitignore`** while implementing the pagination fix for issue `iss-1784874509379-grf5k8` (commit `8416b57`). Both files are listed in `scripts/protected-files.json` but the consumer's `revertAndStageProtected` mechanism failed to intercept these deletions:

- **`.gitignore`** was staged for deletion (`D ` in git status). `git checkout -- .gitignore` fails silently when the file has been removed from the index, and the try/catch in `issue-consumer.mjs:422` swallows the error.
- **`Dockerfile.consumer`** was deleted from the working tree (unstaged, ` D`). The `git checkout --` restoration either failed or the file was already gone before the revert step ran.

### Why the user doesn't see pagination

The pagination code **IS present and correct** in the source:
- `web/src/components/GatewayPanel.tsx:113-330` — client-side pagination with `PAGE_SIZE=10`, prev/next buttons, page indicator
- `web/src/styles.css:3695-3706` — `.pagination` and `.pagination__info` CSS classes
- `server/src/routes/gateway.ts:107-114` and `server/src/db.ts:319-323` — server returns all routes without any LIMIT

The console (web app) likely deployed with the pagination fix. However, the Deploy to EC2 workflow (`.github/workflows/deploy.yml:97-99`) fails at the consumer build step:

```bash
docker compose build --no-cache consumer && docker compose up -d consumer || \
  docker compose up -d consumer
```

Without `Dockerfile.consumer`, both the build and the fallback `docker compose up -d consumer` fail (no `image:` key in docker-compose.yml means Docker Compose must build from the Dockerfile). With `set -e` active, the deploy script exits before executing the issue resolution step (lines 106-131), leaving the deployment in an incomplete state.
