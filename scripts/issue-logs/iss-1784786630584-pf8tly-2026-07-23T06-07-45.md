## Response

## Analysis

**Root cause:** The assistant system prompt was missing awareness of the `dockyard-knowledge` bucket convention. Without this context, the assistant wouldn't automatically check for per-resource knowledge notes before operating on resources, wouldn't factor those notes into its decisions, and wouldn't offer to update them after making changes.

**Status:** This issue has **already been fixed** in commit `41fda4c` ("fix: Add dockyard-knowledge bucket convention to assistant system prompt"), which is the current HEAD of `main`.

**What the fix did:** The commit added a paragraph to the `SYSTEM` constant in `/app/server/src/routes/assistant.ts` at line 100:

> *A knowledge base bucket named `dockyard-knowledge` holds per-resource markdown notes keyed as `{type}/{id}.md` (e.g. `container/ct-abc123.md`, `fn/fn-xyz789.md`). Before operating on any resource, check whether a note exists at the expected key by calling read_bucket_object. If one exists, read it and factor its contents — especially deploy methodology, gotchas, dependencies, and future plans — into every decision you make about that resource. After making meaningful changes to a resource, offer to update its note.*

This text matches the `suggested_prompt_text` from the issue exactly. It is placed after the opening role statement and before the "Rules:" section, making it prominent context that the model will see at the start of every conversation.

The three behaviours called for in the issue are all covered:
1. **Check for notes** — "Before operating on any resource, check whether a note exists at the expected key by calling read_bucket_object."
2. **Factor contents into decisions** — "If one exists, read it and factor its contents — especially deploy methodology, gotchas, dependencies, and future plans — into every decision you make about that resource."
3. **Offer to update after changes** — "After making meaningful changes to a resource, offer to update its note."

**No code changes are needed** — the fix is complete and already deployed on `main`.
