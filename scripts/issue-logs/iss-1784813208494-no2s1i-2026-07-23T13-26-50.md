## Response

Here's a summary of all changes made to fix the issues list compliance and blank detail pages.

---

## Root Causes

1. **Non-standard UI**: The issues list used custom CSS classes (`issues-page`, `issues-card`) and card-based layout instead of the standard `<section className="panel">` / `<table className="table">` pattern used by every other list in the app (Instances, Functions, Buckets, Gateway, Databases).

2. **Missing server route — blank pages**: The server had `DELETE /issues/:id` and `PATCH /issues/:id` but **no `GET /issues/:id`**. Every detail link returned a 404, causing blank/failed pages.

3. **Broken "+ New Issue" button**: It navigated to `/issues/new`, which matched `/issues/:id` with id="new" — also a 404 — instead of opening the existing `CreateIssueModal`.

## Changes Made (7 files)

### `web/src/pages/IssuesList.tsx` — Rewritten to match standard list pattern
- Replaced custom card layout with `<section className="panel">` + `<table className="table">` with `<thead>` and `<tr>` rows
- Added `<span className="count">` showing total issue count
- "+ New Issue" now calls `onCreateIssue` prop (opens `CreateIssueModal`) instead of navigating to broken `/issues/new`
- Added `onRefresh` subscription so the list auto-reloads when the assistant mutates issues
- Replaced local `timeAgo()` with the shared `timeAgo` from `../format`

### `web/src/pages/IssueDetail.tsx` — Fixed blank pages
- Wrapped content in `<section className="panel">` for standard look
- Added `onRefresh` subscription for auto-reload
- Cleaned up async loading pattern, removed unnecessary `Promise.all`

### `server/src/routes/assistant.ts` — **Added missing `GET /issues/:id` route** (root cause of blank pages)
- New route between `GET /issues` and `POST /issues` that calls `getAssistantIssue()`, returns `toIssueSummary()` on success, 404 otherwise

### `web/src/api.ts` — Added missing type parameters
- `assistantListIssues`: now returns `Promise<AssistantIssue[]>` (was `Promise<unknown>`)
- `assistantGetIssue`: now returns `Promise<AssistantIssue>` (was `Promise<unknown>`)

### `web/src/App.tsx` — Passed modal control to issues page
- Issues route now passes `onCreateIssue={() => setCreateIssueOpen(true)}`

### `web/src/pages/Home.tsx` — Made issues card consistent
- "Open issues" card changed from plain `<div>` to clickable `<button>` navigating to `/issues`

### `web/src/styles.css` — Removed dead CSS
- Removed `.issues-page__head`, `.issues-card*` (5 rules), `.issue-detail` bare class — all replaced by standard panel/table classes
