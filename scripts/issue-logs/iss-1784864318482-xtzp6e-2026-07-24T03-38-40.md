## Response

## Summary

**Root Cause:** The grouped session headers (Today, Yesterday, This Week, Earlier) rendered in the sidebar were static `<div>` elements — non-interactive labels. All sessions from every group were always visible, defeating the purpose of grouping because the list couldn't be collapsed to focus only on recent sessions.

**Diagnosis:** Commit `72fc3a1` added `groupSessionsByDate()` to group sessions into date buckets and render group headers, but didn't add expand/collapse interactivity. The headers were plain text labels with no click handler, no chevron indicator, and no conditional rendering of the items beneath each group.

**Changes:**

1. **`web/src/App.tsx`** — Added collapsible group behavior:
   - New `collapsedSessionGroups` state, initialized with `Yesterday`, `This Week`, and `Earlier` collapsed (only "Today" starts open)
   - New `toggleSessionGroup(label)` function to toggle a group's collapsed state
   - Group headers are now clickable (`role="button"`, `onClick`, keyboard support via `onKeyDown`) with a directional chevron (`▸`/`▾`) and item count
   - Group items are conditionally rendered only when the group is not collapsed

2. **`web/src/styles.css`** — Styled the interactive group header:
   - Added `cursor: pointer`, `user-select: none`, hover background transition, and `display: flex` for the chevron/count layout
   - Added `.offcanvas-session-group-chevron` and `.offcanvas-session-group-count` rules
